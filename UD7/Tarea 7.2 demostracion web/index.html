4. Indica los eventos estándares en el DOM

De ratón

Algunos de ellos son:

  • click: hacer click en un elemento
  • dblclick: al hacer doble click en un elemento
  • mousedown: al pulsar sobre un elemento
  • mouseup: al levantar el botón después de hacer mousedown
  • mouseover: al poner el cursor sobre un elemento
  • mousemove: mientras el cursor está pasando sobre un elemento
  • mouseout: al salir el cursor de un elemento

De teclado

Algunos de ellos son:

  • keydown: al pulsar una tecla, aparece antes del keypress.
  • keypress: al presionar una tecla (se mantiene), aparece después del keydown.
  • keyup: al dejar de presionar una tecla

De HTML:

Algunos de ellos son:

  • load: cuando se ha cargado todo el elemento, incluyendo frames, imagenes, etc.
  • unload: cuando el user agent ha eliminado todos los elementos de la ventana o frame
  • error: cuando una imagen u objeto o frame no se carga correctamente
  • abort: cuando se detiene la carga de una imagen u objeto o frame antes de cargarse completamente

De formulario

Algunos de ellos son:

  • select: al seleccionar un texto en un campo de texto (incluyendo tipo text y textarea).
  • focus: al recibir el foco un elemento del formulario
  • blur: al perder el foco un elemento del formulario
  • change: cuando un input pierde el foco y su valor ha cambiado
  • submit: Al hacer un sumit del formulario
  • reset: Al hacer reset en el formulario

6. Diferencia entre .on(), .live(). bind() y .delegate()

bind() (Está obsoleto)

Se utilizaba para definir cualquier tipo de evento, pudiendo definir incluso varios tipos de eventos a la vez. Solo funcionaba para los elementos coincidentes con el selector existente en el momento de la ejecución del método bind. Internamente utilizaba el método on.

$("a").bind("click", function(event){ //aquí haríamos algo });

live() (Está obsoleto)

A diferencia de bind, si permitía definir los eventos tanto para elementos que en ese momento coinciden con el selector pero también para los que pertenecerán a él en el futuro. Internamente utilizaba el método on.

delegate() (Está obsoleto)

Asignaba una función manejadora a uno o varios eventos para los elementos que coincidiesen con el selector, tanto para elementos que en ese momento coinciden con el selector pero también para los que pertenecerán a él en el futuro, y que se encuentren dentro del contexto dado.

$("aside").delegate("a","click", function(event) { //aquí haríamos algo });

La ventaja de este método con respecto al anterior era una mayor especificidad y un recorrido en el DOM más eficiente al no tener que recorrer el árbol completo.

on()

Este método define una función ante uno o más eventos para los elementos seleccionados. Es lo que realmente utiliza internamente jquery con otros métodos como click. Se puede utilizar para lo que hacíamos tanto con bind como con delegate.

$("div").on("a","click", function(event) { //aquí haríamos algo }); $("a").on("click", function(event) { //aquí haríamos algo });
